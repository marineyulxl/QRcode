# 02 — 出货单双角色扫码（方案 + Mock 实现）

本文档与 [`01-来访登记三页静态站.md`](01-来访登记三页静态站.md) 一样：**一个模块一篇**。

---

## 一、业务概要（当前口径）

- **司机扫码**：进入司机页；页面请求后端 **最近一条** 车辆识别结果（车牌 + 车辆图，只读），再必填驾驶人身份资料与人脸照片并提交。
- **库管扫码**：进入库管页；**车牌须手动选择**（默认「请选择」），选定后展示该车辆照片及已关联的司机登记信息；填报 **物资名称、数量**、**出库类别**（界面仅展示类别名称：检修 / 防腐 / 短节；责任人由后端按类别映射）；**不设库管现场拍照**；提交出库登记。
- **数据**：司机段 + 库管段 + 最近识别车辆信息在同一会话/后端单据中合并（Mock 用单一 `sessionStorage` 快照）。

## 二、双二维码（仍保留两个入口）

| 角色 | 路径 | 二维码示例 URL |
|------|------|----------------|
| 司机 | `/out/driver` | `https://域名/out/driver` |
| 库管 | `/out/warehouse` | `https://域名/out/warehouse` |

两条链接区分角色，扫码直达；URL 不写业务单号，由后端返回识别记录与在场车辆列表。

## 三、暂时没有后端时（边界）

- **做不到**：真实「最近一条」与海康联动、可靠的多端合并。
- **当前仓库 Mock**：司机进入页拉取快照里的「最近一条」模拟数据；按钮「刷新最新识别」模拟换新的一条识别（并清空当前司机填报）；库管下拉为固定演示车牌池 + 若本浏览器已有司机提交则置顶该车。

## 四、已实现（Mock）

### 路由

| 页面 | 路径 |
|------|------|
| 司机 | `/out/driver` |
| 库管 | `/out/warehouse` |

### 司机页

- 自动请求 **最近一条** Mock 海康数据（车牌 + 图）。
- 「刷新最新识别」：轮换演示车牌，清空当前快照中的 **司机填报**（模拟新车）。
- 提交写入 `sessionStorage` 键 `outbound_latest_snapshot`。

### 库管页

- **Mock 车牌列表**：默认包含与司机页同一套演示车牌及其虚构司机信息（便于未连后端时直接验收）；若本机快照里已有司机提交且车牌与某条演示一致，则 **以快照覆盖该车**；若提交了演示池以外的车牌，则 **插在列表最前**。
- 默认不预选，选定后展示车辆照片与司机登记信息（数据随列表项一并返回）。
- **同步列表**：重新读取当前 Mock 列表（接后端后对应在场车辆接口）。
- 出库类别仅「检修 / 防腐 / 短节」；`assignee` 不展示在界面。
- 无库管拍照字段；提交后控制台打印完整快照。

### 演示顺序

1. **可只打开库管页**：`/out/warehouse` 直接选演示车牌即可填报（Console 仍打印合并快照）。  
2. **完整链路**：`/out/driver` 提交司机关联某车牌 → `/out/warehouse` 点「同步列表」→ 该车以登记数据覆盖同牌演示行。  
3. 库管提交后看 Console：`[OutboundMock] warehouse submitted full snapshot`。

## 五、正式上线（与后端对齐）

- 司机页：`GET …/outbound/latest-recognition`（示例）返回最近一条车牌与车辆图。
- 库管页：`GET …/vehicles/on-yard` 等返回下拉列表；选中后带出司机信息由后端关联。
- 提交：`POST` 各自接口；**sessionStorage 方案作废**。
- 海康密钥仅服务端持有。
