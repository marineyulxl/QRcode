# 02 — 出货单双角色扫码（方案 + 前后端对接）

本文档与 [`01-来访登记三页静态站.md`](01-来访登记三页静态站.md) 一样：**一个模块一篇**。

---

## 一、业务概要（当前口径）

- **司机扫码**：进入司机页；页面请求后端 **最近一条** 车辆识别结果（车牌 + 车辆图，只读），再必填驾驶人身份资料与人脸照片并提交。
- **库管扫码**：进入库管页；**车牌须手动选择**（默认「请选择」），选定后展示该车辆照片及已关联的司机登记信息；填报 **物资名称、数量**、**出库类别**（界面仅展示类别名称：检修 / 防腐 / 短节；责任人由后端按类别映射）；**不设库管现场拍照**；提交出库登记。
- **数据**：司机建档与库管补录由后端 **`back-vehicle`** 单据承载；司机页「识别条」仍为前端 Mock，仅作演示。

## 二、双二维码（仍保留两个入口）

| 角色 | 路径 | 二维码示例 URL |
|------|------|----------------|
| 司机 | `/out/driver` | `https://域名/out/driver` |
| 库管 | `/out/warehouse` | `https://域名/out/warehouse` |

两条链接区分角色，扫码直达；URL 不写业务单号，由后端返回识别记录与在场车辆列表。

## 三、暂时没有真实海康时（边界）

- **做不到**：司机页展示真实道闸「最近一条」识别。
- **当前仓库**：司机页识别区仍为 **`outbound-mock`**（`sessionStorage` 单条快照）；提交不带上识别字段。

## 四、当前实现

### 路由

| 页面 | 路径 |
|------|------|
| 司机 | `/out/driver` |
| 库管 | `/out/warehouse` |

### 司机页

- **车辆识别**：仍为 **`outbound-mock`** 演示（车牌 + 占位图 +「刷新识别结果」），**不参与**真实提交 Body。
- **提交**：`POST /admin-api/camera/back-vehicle/create`，先 `POST /admin-api/infra/file/upload` 上传自拍，Body 仅 `name`、`idCard`、`phone`、`selfiePhoto`（见 [`src/api/back-vehicle.ts`](../../src/api/back-vehicle.ts)）。成功提示：`data` 为可读字符串则用 `data`，否则 `msg` 非空用 `msg`，`data` 为数字 id 时前端固定话术（见 `pickCreateSuccessMessage`）。
- **不再写入** Mock 快照（库管列表走后端 `page` 接口）。

### 库管页

- **列表**：`GET /admin-api/camera/back-vehicle/page`，无额外 Query（若后端支持筛选，可按接口文档传 `name` / `phone` / `license` 等）；响应 `{ total, list }`，字段含 `itemName` / `quantity` / `type`（未填为 `null`）、`approvalOpinion`（`0` 未审批等）。
- 前端下拉仅展示 **`approvalOpinion === '0'`** 的待库管补录行（见 `pendingLibraryRows`）；选项 **`id`** + 展示 **`license`**。
- **同步列表**：重新请求上述接口。
- **提交**：`PUT /admin-api/camera/back-vehicle/library_update`，Body 含当前行全量展示字段 + 库管填报的 `itemName`、`quantity`（字符串）、`type`（中文：检修 / 防腐 / 短节，与 [`outbound-category-types`](../../src/constants/outbound-category-types.ts) 一致）。
- 出库类别界面不展示 `assignee`；无库管拍照字段。

### 演示顺序（连后端时）

1. 司机在 `/out/driver` 完成 **`back-vehicle/create`**（自拍 + 身份信息）。  
2. 库管打开 `/out/warehouse`，点「同步列表」，选中对应 **`license`**，补录物资并提交 **`library_update`**。
