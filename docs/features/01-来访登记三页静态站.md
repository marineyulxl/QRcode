# 01 — 来访登记三页静态站（已实现）

## 目标

移动端扫码进入：**入口二选一**（人员登记 / 车辆登记），表单必填校验与拍照。人车均已对接后端：**先上传图片拿到 URL，再分别调用人员 / 车辆建档接口**。

## 路由

| 路径 | 页面 |
|------|------|
| `/` | 入口双按钮 [`src/views/entry-page.vue`](../../src/views/entry-page.vue) |
| `/person` | 人员登记 [`src/views/person-register.vue`](../../src/views/person-register.vue) |
| `/vehicle` | 车辆登记 [`src/views/vehicle-register.vue`](../../src/views/vehicle-register.vue) |

路由定义：[ `src/router/index.ts`](../../src/router/index.ts)。

## 人员登记：后端接口约定

| 步骤 | 方法 | 路径（相对 `apiBaseUrl`，默认 `/admin-api`） | 说明 |
|------|------|---------------------------------------------|------|
| 上传人脸 | POST | `/infra/file/upload` | `multipart/form-data`，字段名 `file`；返回文件访问地址，写入下一步 `picture` |
| 提交人员 | POST | `/camera/front-personnel/create` | JSON  body，见下表 |

**请求头（两步均需）**

- `tenant-id`：当前固定 `1`（见 [`src/config/env.ts`](../../src/config/env.ts)）。

**`POST /camera/front-personnel/create` 请求体**

| 字段 | 类型 | 对应表单 |
|------|------|----------|
| `name` | string | 访客姓名 |
| `idCard` | string | 身份证号 |
| `phone` | string | 手机号 |
| `picture` | string | 人脸照片 URL（上传接口返回的字符串） |
| `number` | string | 随访人数；未填时传 `"0"` |
| `subject` | string | 事由 |

**本地直连示例**（不经前端代理时）：`http://127.0.0.1:48080/admin-api/camera/front-personnel/create`。

**响应约定**：与常见 `{ code, data, msg }` 一致，`code` 为 `0` 或 `200` 视为成功（axios 实例见 [`src/api/request.ts`](../../src/api/request.ts)）。

若实际上传路径、表单字段名或返回结构与若依默认不一致，改 [`src/api/infra-file.ts`](../../src/api/infra-file.ts) 的路径或解析逻辑，并在本文档同步。

## 车辆登记：后端接口约定

| 步骤 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 车辆类型 | GET | `/camera/vehicle-type/page` | 无参数；`data.list[]` 为 `{ id, content }`，下拉 value=`id`（[`front-vehicle.ts`](../../src/api/front-vehicle.ts)） |
| 上传司机自拍 | POST | `/infra/file/upload` | 同人员，`file` 字段 |
| 上传车辆照片 | POST | `/infra/file/upload` | 第二次上传 |
| 提交车辆 | POST | `/camera/front-vehicle/create` | JSON，见下表 |

**`POST /camera/front-vehicle/create` 请求体**

| 字段 | 对应表单 |
|------|----------|
| `name` | 司机姓名 |
| `idCard` | 身份证号 |
| `phone` | 手机号 |
| `license` | 车牌号（大写） |
| `type` | 车辆类型：分页接口项的 `id`（字符串） |
| `selfiePhoto` | 司机自拍 URL |
| `vehiclePhoto` | 车辆照片 URL |

成功提示逻辑与人员接口一致：`data` 为字符串时优先作 Toast。

## 主要实现要点

- **技术栈**：Vite + Vue 3 + TypeScript + Vue Router + Tailwind v3。
- **别名**：`#/*` → `src/*`（[`vite.config.ts`](../../vite.config.ts)、[`tsconfig.app.json`](../../tsconfig.app.json)）。
- **样式主题**：[`tailwind.config.js`](../../tailwind.config.js)（Minimalism / 访客场景 token 注释在文件头）。
- **拍照**：[`src/components/capture-photo-field.vue`](../../src/components/capture-photo-field.vue)，`capture` 区分前置/环境；[`src/utils/image-compress.ts`](../../src/utils/image-compress.ts) 压缩。
- **校验**：[`src/utils/validation.ts`](../../src/utils/validation.ts)（手机、身份证、车牌、随访人数；入参做字符串归一化避免非 string）。
- **HTTP**：[`src/api/request.ts`](../../src/api/request.ts)；上传 [`src/api/infra-file.ts`](../../src/api/infra-file.ts)；人员 [`src/api/front-personnel.ts`](../../src/api/front-personnel.ts)；车辆建档与类型分页 [`src/api/front-vehicle.ts`](../../src/api/front-vehicle.ts)。
- **全局轻提示**：[`src/utils/toast-bus.ts`](../../src/utils/toast-bus.ts) + [`src/App.vue`](../../src/App.vue)。
- **布局**：顶栏 / 底栏 `safe-area-inset`；底部固定提交按钮。

## 产品取舍记录

- 曾有人车两侧「海康」勾选，已 **移除**：含义改为「整单提交，是否对接海康由后端决定」。
- 随访人数不用 `type="number"` 的 `v-model`，避免 Vue 输出 number 导致 `.trim` 报错。

## 部署与二维码

见仓库根目录 [`DEPLOY.md`](../../DEPLOY.md)（含宝塔、`try_files`、HTTPS）。二维码内容即 **线上可访问的首页 URL**（建议 HTTPS）。

生产环境请求仍走 **同源 `/admin-api`**，由 Nginx 反代到后端；**不使用** Vite `server.proxy`。
