# 01 — 来访登记三页静态站（已实现）

## 目标

移动端扫码进入：**入口二选一**（人员登记 / 车辆登记），表单必填校验与拍照，提交为 **纯前端占位**（无真实后端）。

## 路由

| 路径 | 页面 |
|------|------|
| `/` | 入口双按钮 [`src/views/entry-page.vue`](../../src/views/entry-page.vue) |
| `/person` | 人员登记 [`src/views/person-register.vue`](../../src/views/person-register.vue) |
| `/vehicle` | 车辆登记 [`src/views/vehicle-register.vue`](../../src/views/vehicle-register.vue) |

路由定义：[ `src/router/index.ts`](../../src/router/index.ts)。

## 主要实现要点

- **技术栈**：Vite + Vue 3 + TypeScript + Vue Router + Tailwind v3。
- **别名**：`#/*` → `src/*`（[`vite.config.ts`](../../vite.config.ts)、[`tsconfig.app.json`](../../tsconfig.app.json)）。
- **样式主题**：[`tailwind.config.js`](../../tailwind.config.js)（Minimalism / 访客场景 token 注释在文件头）。
- **拍照**：[`src/components/capture-photo-field.vue`](../../src/components/capture-photo-field.vue)，`capture` 区分前置/环境；[`src/utils/image-compress.ts`](../../src/utils/image-compress.ts) 压缩与 `fileToDataUrl`。
- **校验**：[`src/utils/validation.ts`](../../src/utils/validation.ts)（手机、身份证、车牌、随访人数；入参做字符串归一化避免非 string）。
- **占位 API**：[`src/api/visitor-mock.ts`](../../src/api/visitor-mock.ts) — `VisitorApi.submitPerson` / `submitVehicle`。
- **全局轻提示**：[`src/utils/toast-bus.ts`](../../src/utils/toast-bus.ts) + [`src/App.vue`](../../src/App.vue)。
- **布局**：顶栏 / 底栏 `safe-area-inset`；底部固定提交按钮。

## 产品取舍记录

- 曾有人车两侧「海康」勾选，已 **移除**：含义改为「整单提交，是否对接海康由后端决定」。
- 随访人数不用 `type="number"` 的 `v-model`，避免 Vue 输出 number 导致 `.trim` 报错。
