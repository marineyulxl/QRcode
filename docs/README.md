# 项目文档与回溯目录

本目录用于 **记录做过哪些功能、怎么做、关键文件在哪**，方便日后回溯、交接和对照 Cursor / 本地计划。

## 怎么用

| 类型 | 路径 | 说明 |
|------|------|------|
| 功能说明（按模块） | [`features/`](features/) | 每个文件一篇：`做了什么`、`文件清单`、`配置与部署`、`已知取舍` |
| 按时间流水账 | [`CHANGELOG.md`](CHANGELOG.md) | 简短日期条目：改了啥、关联哪个 feature 文档 |

**约定：**新做了一整块功能时：

1. 在 `features/` 下新增 `NN-功能简述.md`（`NN` 为两位序号，递增）。
2. 在 `CHANGELOG.md` 顶部追加一条，指向对应 feature 文档。

Cursor 里的 `.cursor/plans/` 计划可与本文档 **互相引用**：落地后把结论摘要同步进 `features/`，避免只在聊天记录里。

## 与仓库根目录其他文档的关系

- [`../DEPLOY.md`](../DEPLOY.md)：静态部署、宝塔、Nginx SPA、HTTPS 等运维说明。
- 本 `docs/`：**产品与实现回溯**，偏「人读」的需求与实现备忘。
